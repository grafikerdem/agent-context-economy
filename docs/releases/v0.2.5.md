# Agent Context Economy v0.2.5

v0.2.5 adds explainable context reduction to the existing ACE methodology. Compact helper output now states what was evaluated, what was excluded, why the returned context was selected, and which smallest next action can recover missing evidence.

## Highlights

- Adds a bounded `PROVENANCE` footer to seven PowerShell helpers.
- Reports repository and Git identity with graceful `unknown` values when Git context is unavailable.
- Reports scope, exclusions, considered and returned counts, reduction state, selection reason, and next step.
- Preserves existing helper parameters and primary output sections.
- Extends smoke tests with provenance marker checks.
- Regenerates benchmark artefacts with provenance overhead included.

## Relationship to the ACE Stack

Explainable context reduction is not a seventh layer. It is a cross-cutting quality applied to compacting helpers across Discovery, Targeted Reading, and Validation. The six-layer workflow and `docs/workflow.svg` therefore remain unchanged.

## Helpers with Provenance

- `run-compact.ps1`
- `search.ps1`
- `investigate.ps1`
- `read-window.ps1`
- `read-symbol.ps1`
- `diff-summary.ps1`
- `diff-file.ps1`

Each footer is limited to 8–12 lines and answers the audit questions needed to understand a compact result without recreating the full raw context.

## Compatibility

- Windows PowerShell 5.1 or later
- no external package dependencies
- no Node.js, Python, Rust, WSL, or Docker requirement
- existing script parameters remain backward compatible
- generated writes remain under `.agent-context` or `benchmark-results`

## Verification

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\powershell\smoke-test.ps1
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\powershell\benchmark.ps1
```

Verified benchmark results:

- 91% less terminal output
- 87% less source code read
- 71% fewer shell commands

## Known Limits

- Search and investigation totals are sampled totals when configured limits are reached.
- Symbol navigation remains regex-based rather than AST-based.
- Provenance resolves repository and Git identity from the current working directory.
- Existing ripgrep parsing of Windows absolute paths can display the drive prefix as the file name; v0.2.5 does not change search result parsing.
