# ACE v1.0.0 - Stable Framework Release

We are proud to announce the **v1.0.0 stable release of Agent Context Economy (ACE)**! This milestone marks the maturity of ACE from a collection of helper scripts into a formalized, production-ready, cross-platform **LLM-first progressive source navigation framework**.

---

## What is ACE?
ACE (Agent Context Economy) is a cross-platform, LLM-first progressive source navigation framework that helps autonomous coding agents retrieve only the code required for the current reasoning step. It prevents token pollution, cuts context bills, and keeps agents focused by implementing the progressive retrieval model:
```
Discovery → Summary → Signature → Body → Bounded Window
```

---

## Core Features of v1.0.0
- **Progressive Retrieval**: Tiers of code exposure designed for LLMs:
  - `Summary`: Outlines containers, functions/methods, and exports/variables.
  - `Signature`: Hides implementation blocks to show only public contracts.
  - `Body`: Extracts specific executable scopes.
  - `Full`: Bounded raw file reading.
- **Context Budgeting**: Abstract token limits (`-Budget Small | Medium | Large`) allowing automatic, constraint-based output calculation.
- **Adaptive Guidance**: Autonomously directs agents on the next logical command under the `Next:` provenance footer.
- **Smart Truncation**: Intelligently truncates large method blocks to keep definitions, starts, and ends visible while omitting filler blocks.
- **Modular ACE Libraries**: Consolidates parser, formatting, and truncation helpers into side-effect-free shared PowerShell modules under `lib/`.

---

## Cross-Platform Support
ACE v1.0.0 is fully cross-platform, powered by **PowerShell 7 (`pwsh`)** core runtime compatibility:
- **Windows**: Native PowerShell 5.1/7 compatibility.
- **macOS / Linux**: Dedicated, robust thin Bash wrapper scripts (`bin/ace-read-symbol`, `bin/ace-read-text`, `bin/ace-search`) that execute seamlessly on Unix-like filesystems.

## Stable API Specification

To ensure backwards compatibility, the following API elements are declared **stable** starting in v1.0.0:

### 1. Progressive Reading Switches
- **`-Summary`**: Outline mode.
- **`-Signature`**: Interface/contract-only mode.
- **`-Body`**: Implementation block extraction.
- **`-Full`**: Full raw source.

### 2. Context Budgets
- **`-Budget Small`** (~400 tokens / 30 lines)
- **`-Budget Medium`** (Default, ~1200 tokens / 100 lines)
- **`-Budget Large`** (~3000 tokens / 250 lines)

### 3. Output Formats
- **Metadata Header**: Standardized headers (`=== SELECTED SYMBOL WINDOW ===` and `=== UTF-8 TEXT READ ===`).
- **Guidance Footer**: Standardized provenance metadata block including `Next:` instructions for agent navigation.
