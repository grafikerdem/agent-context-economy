# Agent Context Economy v0.2.0

v0.2.0 evolves Agent Context Economy from a set of PowerShell helpers into a methodology and reference implementation for context-efficient AI software development.

## Highlights

- Introduces the six-layer Agent Context Economy Stack.
- Adds repository orientation with `repo-map.ps1`.
- Adds small, disposable continuity state with `session-state.ps1`.
- Adds compact startup briefings with `agent-start.ps1`.
- Documents why ACE exists and how it works with MCP servers and other discovery engines.
- Adds generated Markdown, CSV, JSON, badge, and SVG benchmark artefacts.
- Expands the example agent policy with startup, fallback, and approval-economy rules.

## The Stack

```text
Continuity
  -> Repository Map
  -> Discovery
  -> Targeted Reading
  -> Workflow
  -> Validation
```

The workflow is tool-agnostic. The included scripts provide a dependency-free Windows PowerShell reference implementation.

## Compatibility

- Windows PowerShell 5.1 or later
- no external package dependencies
- no Node.js, Python, Rust, WSL, or Docker requirement
- explicit UTF-8 output for generated files

## Upgrade Notes

- Existing helper commands remain available under `scripts/powershell`.
- Run `repo-map.ps1` once when beginning work in a repository or after a material structural change.
- Treat `.agent-context/session-state.json` as disposable local metadata; it is excluded from Git.
- Copy the relevant new policies from `examples/AGENTS.example.md` into the target repository's agent instructions.
