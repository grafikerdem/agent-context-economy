# Release Notes - v0.2.6 (Progressive Navigation Edition)

We are proud to announce the **v0.2.6 release of Agent Context Economy (ACE)**! This milestone focuses on transition from simple source dumping to structured, progressive information retrieval designed specifically to prevent model context flooding.

---

## What's New in v0.2.6

### 1. Modular PowerShell Architecture
To keep the codebase maintainable and dry, duplicate helper routines were extracted into side-effect-free shared library scripts under `scripts/powershell/lib/`:
- **`ACE.Parser.ps1`**: Symbol normalizations, character counts, and recursive brace-depth matching algorithms.
- **`ACE.Formatting.ps1`**: Unified metadata headers, provenance footer generation, and token budget resolvers.
- **`ACE.Truncation.ps1`**: Disjoint smart truncation calculations.

### 2. Progressive Reading Modes
Code navigation tools now support progressive retrieval tiers rather than all-or-nothing file reading:
- **`Summary`**: Outputs high-level outlines (containers, functions/methods, and exports/variables) with line numbers.
- **`Signature`**: Strips implementation blocks to show only public contracts (maximum 5 lines per signature).
- **`Body`**: Isolates implementation braces (`{ ... }`) for methods/functions.
- **`Full`**: Returns raw source with progressive dynamic context boundaries.

### 3. Context Budgeting (`-Budget`)
Instead of guessing lines, agents can now specify abstract token constraints:
- **`-Budget Small`** (~400 tokens / 30 lines limit)
- **`-Budget Medium`** (default, ~1200 tokens / 100 lines limit)
- **`-Budget Large`** (~3000 tokens / 250 lines limit)

### 4. Adaptive Next Command Recommendations
To assist autonomous agents in navigating without guesswork, both tools analyze their result state and print the next logical escalation path under the `Next:` provenance footer or guidance block.

---

## Upgrade & Migration Guide

- **`read-text.ps1` Default Change**: Sourcing a code file without parameters now automatically prints its **Summary** (Outline index) instead of raw text. To print full contents, explicitly add the `-Full` switch.
- **`read-symbol.ps1` Default Change**: Symmetrical defaults are now applied: containers (classes/types) default to **Summary**, while executables (functions/methods) default to **Signature**.
- **Libraries Sourcing**: Sourced modules under `lib/` are guaranteed to produce zero stdout side effects during import.
